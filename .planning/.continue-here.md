<decisions_made>

- **Context injection approach**: Instead of a separate context block the GLM could ignore, inject context directly adjacent to the query line (`Query: "how many in Bengaluru"\n\nCONTEXT ‚Äî previous was "list murder cases"...`)
- **query_context format**: Store as `sql ‚Äî explanation` instead of just explanation so follow-up can see the actual WHERE clauses
- **District LIKE matching**: Use `d.DistrictName LIKE '*Bengaluru*'` instead of exact match since users type partial names
- **LIKE wildcard**: Use `*` (ZCQL V2) not `%` (standard SQL)
</decisions_made>
