## Deploying Functions

### Prerequisites

- [Catalyst CLI](https://www.zoho.com/catalyst/install.html) installed
- Logged in via `catalyst login` with appropriate project access
- `.catalystrc` present in the project root (auto-generated by `catalyst init`)

### Deploy All Functions

```bash
catalyst deploy
```

This deploys every target listed in `catalyst.json` — all 13 functions plus the client slate. The CLI reads the project ID from `.catalystrc` and pushes source from `functions/{name}/` for each target.

## Critical: Environment Variable Wipe

**`catalyst deploy` overwrites Console environment variables.** After every deploy, `QUICKML_TOKEN` (the GLM OAuth Bearer token) must be re-added through Catalyst Console for the four functions that call the GLM LLM.

| Function | Requires `QUICKML_TOKEN` | Requires `CATALYST_ORG` |
|----------|--------------------------|------------------------|
| `classifier` | Yes | In code fallback |
| `nl_sql` | Yes | In code fallback |
| `rag` | Yes | In code fallback |
| `pipeline` | Yes | In code fallback |

The token value is a Self Client OAuth token (format: `1000.{hash}.{hash}`) generated from the Zoho API Console with scope `QuickML.deployment.READ`. It expires every 60 minutes.

<!-- VERIFY: The actual QUICKML_TOKEN value must be obtained from Zoho API Console > Self Client — it is not stored in the repository -->

### `CATALYST_ORG` — Reserved Keyword

`CATALYST_ORG` is a **reserved Catalyst environment variable** — it cannot be set in `catalyst-config.json`'s `env_variables` block. All four functions that need it handle it with a code fallback:

```javascript
const CATALYST_ORG = process.env.CATALYST_ORG || '60073929329';
```

The value `60073929329` is the organization ID. If the Console environment variable is set, it overrides the default.

<!-- VERIFY: The org ID `60073929329` was discoverable from `.catalystrc` — confirm it is still active if project settings have changed -->

### Adding Environment Variables via Catalyst Console

After each deploy, re-add `QUICKML_TOKEN` manually for each of the four functions:

1. Open **Catalyst Console** → select the project (Datathon2026)
2. Navigate to **Functions** → select the function (e.g., `pipeline`)
3. Go to the **Environment Variables** tab
4. Click **Add Variable**
5. Set **Key:** `QUICKML_TOKEN`, **Value:** `{current token from Zoho API Console}`
6. Click **Save**
7. Repeat for each function: `classifier`, `nl_sql`, `rag`, `pipeline`

<!-- VERIFY: The Catalyst Console URL path varies per region — use https://console.catalyst.zoho.in or your region's Catalyst Console URL -->

### Future: Auto-Refresh OAuth (Server-based App)

The current Self Client OAuth flow requires manual token refresh every hour. A planned migration to Server-based App OAuth (see `docs/production-auth.md`) will eliminate this with `refresh_token` auto-renewal. The required env vars for that migration are:

| Variable | Purpose |
|----------|---------|
| `QUICKML_CLIENT_ID` | Server-based App client ID |
| `QUICKML_CLIENT_SECRET` | Server-based App client secret |
| `QUICKML_REFRESH_TOKEN` | Never-expiring refresh token |

These are **not yet implemented** in the function code — currently only `QUICKML_TOKEN` is consumed.

## Deploying the Data Pipeline (CSV Import)

The synthetic data pipeline lives in `data_pipeline/` and uses the Catalyst CLI `ds:import` command to upload CSV data to Catalyst Data Store tables.

### Generate CSVs and Import

```bash
# From the data_pipeline directory
cd data_pipeline
npm install
node run_phase.js phase1
```

This runs the phase orchestrator which:

1. Generates CSV files via `src/generators/{name}.js` (16 generators for lookup tables in phase 1)
2. Writes to `data_pipeline/data/{FILE_NAME}.csv`
3. Imports each CSV using: `catalyst ds:import {csvPath} --table {TABLE_NAME}`
4. Polls for import completion via `catalyst ds:status write {jobId}`
5. Generates ROWID mapping files in `data_pipeline/mappings/`

### Full Data Pipeline Phases

The pipeline is designed for multi-phase execution (phase 1 = independent lookup tables). Each phase generator file maps to a Catalyst Data Store table:

- `state.js` → `State` table
- `district.js` → `District` table
- `crimeHead.js` → `CrimeHead` table
- `act.js` → `Act` table
- Plus 12 more lookup tables

Legacy phase scripts (`generate_phase*.cjs`) are available under `data_pipeline/` but the current workflow uses `run_phase.js`.

### CSV Export Utilities

Export configurations live in `data_pipeline/export_configs/` — one JSON file per Data Store table (e.g., `CaseMaster.json`, `Accused.json`). These are used with the Catalyst CLI export feature:

```bash
catalyst ds:export {table_name} --config {export_config_path}
```

## Troubleshooting

### `returnErrorResponse` 500

**Root cause:** Corrupted Catalyst Console function registration. This typically happens after repeated deploys or configuration changes.

**Resolution (delete → redeploy → re-add env vars):**

1. Open **Catalyst Console** → **Functions** → select the failing function
2. **Delete** the function from the Console
3. Redeploy the function:
   ```bash
   catalyst deploy --only "functions:{name}"
   ```
4. **Immediately** re-add `QUICKML_TOKEN` via Console (see [Adding Environment Variables via Catalyst Console](#adding-environment-variables-via-catalyst-console))
5. Test the function

### Function Timeout

If a function returns no response after ~30 seconds:

1. Check the 30-second Catalyst hard limit
2. Review GLM timeout settings — GLM requests should have 15–20s timeout
3. Reduce `max_tokens` (200–300 for SQL gen) to speed up GLM responses
4. Check Catalyst Console → Functions → {name} → Logs for "Execution started" messages without completion (these indicate timeouts)

### Env Var Not Found

If a function throws `QUICKML_TOKEN not configured`:

1. Confirm `QUICKML_TOKEN` was re-added via Console after the last deploy
2. Verify the token value is correct (not expired)
3. Generate a fresh Self Client token from Zoho API Console:
   - Go to https://api-console.zoho.com
   - Self Client → scope: `QuickML.deployment.READ` → Generate
   - Copy the token and add it to Console

### GLM Token Expired

Self Client OAuth tokens expire every 60 minutes. If a function that was working suddenly fails:

1. Generate a new token from Zoho API Console
2. Update `QUICKML_TOKEN` in Catalyst Console for all four functions
3. (Long-term: migrate to Server-based App OAuth per `docs/production-auth.md`)
